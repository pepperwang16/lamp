Final Top 3 Picks

1. üåø MushLite Mini ‚Äî Best for simplicity & early user testing

Why: One core interaction: tap to switch modes, tilt to control brightness. Easy to build with one puck LED and foam shell. Perfect for Wizard-of-Oz Pomodoro timer test.

Pomodoro use:
Warm breathing glow for focus
Cool pulsing glow for break
Soft ambient pattern for weather

Ideal for: Initial prototype and usability baseline.

Rational: The MushLite Mini concept focuses on minimal, intuitive interaction and a compact, portable form factor. Its simple cylindrical body and rounded top allow users to control all functions with just tap (to switch modes) and tilt or rotation (to adjust brightness). This design minimizes mechanical complexity while offering clear visual feedback through soft breathing and pulsing light patterns. It‚Äôs ideal for simulating Pomodoro focus/break transitions and simple weather forecast animations, making it an excellent candidate for the first low-fidelity prototype and usability testing. Its small size and single-interaction logic make it easy to fabricate and test with Wizard-of-Oz techniques.

2. üåÄ MushLite Orbit ‚Äî Best for interaction + control

Why: Base rotation gives an intuitive timer setting. Tap and tilt are familiar. Base lighting allows progress feedback and weather effects.

Pomodoro use: Rotate base to set focus time. Base light shows progress, dome shows mode. Forecast animation can run around base.

Ideal for: Richer interaction prototype, second concept in testing.

Rational: The MushLite Orbit builds on the core simplicity of MushLite Mini but introduces a rotating base to give users more control over timer settings. This interaction mimics natural physical gestures, allowing users to rotate the base to set focus/break duration while tapping the top to change modes. The dome and base lighting are separated: the dome conveys the current mode (Focus, Break, Weather), and the base displays progress or forecast through light animations. This separation makes the lamp‚Äôs state easier to read at a glance and provides a richer testing ground for evaluating user comprehension of feedback cues. It offers higher interaction fidelity while remaining feasible for low-fidelity prototyping.

3. ‚ú® MushLite Halo ‚Äî Best for elegant feedback separation

Why: Halo ring gives clear visual progress, dome gives mode state. Still simple to build (light ring or simulation). Strong candidate for aesthetic usability testing.

Pomodoro use: Halo fills gradually during focus; Dome color indicates mode; Halo animations overlay weather forecast (e.g., flow for wind, pulses for rain).

Ideal for: Visual communication clarity & user comprehension tests.

Rational: The MushLite Halo variant emphasizes clarity of visual communication by integrating a halo light ring beneath the dome. This ring provides real-time timer progress visualization, while the dome indicates the current mode through color. This clear division allows users to distinguish between mode state and temporal feedback without extra buttons or screens. Its interaction model remains simple‚Äîtap to switch mode, rotate or tilt for brightness‚Äîand use halo-based gestures or an app for timer control. This makes MushLite Halo particularly strong for evaluating how well users interpret light-based cues for Pomodoro phases and weather conditions.
